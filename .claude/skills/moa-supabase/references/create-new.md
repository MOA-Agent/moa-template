# 신규 Supabase DB 구성 계획

개발자가 DB 구성을 결정할 수 있도록 질문하고, 인프라 담당자 확인용 계획 문서를 생성한다.

## 질문 흐름

아래 질문을 하나씩 순서대로 한다.

**Q1.** Supabase 프로젝트를 어떻게 구성할 건가요?
> 선택지:
> - **새 Supabase 프로젝트 생성** — 이 서비스 전용 프로젝트
> - **기존 Supabase 프로젝트에 추가** — 다른 서비스와 같은 프로젝트 공유

**Q2.** 개발/운영 DB를 분리할 건가요?
> 선택지:
> - **분리** — 개발용 프로젝트, 운영용 프로젝트 별도
> - **통합** — 하나의 프로젝트에서 스키마로 구분
> - **미정**

**Q3.** RLS(Row Level Security)를 적용할 건가요?
> 선택지: **예** / **아니오** / **미정**

**Q4.** 이 DB와 연관된 다른 서비스나 프로젝트가 있나요?
> 자유 응답. 없으면 없음.

## 계획 문서 생성

질문이 끝나면 응답 기반으로 `docs/supabase-plan.md`를 생성한다.

```markdown
# Supabase DB 구성 계획

> 작성일: {날짜}
> 작성자: {사용자에게 이름 묻기}
> 상태: 인프라 담당자 확인 대기 중

## 결정 내용

| 항목 | 결정 |
|---|---|
| 프로젝트 구성 | {새 프로젝트 / 기존 프로젝트에 추가} |
| 개발/운영 분리 | {분리 / 통합 / 미정} |
| RLS 적용 | {예 / 아니오 / 미정} |
| 연관 서비스 | {있으면 목록 / 없음} |

## 미결 항목
- [ ] 인프라 담당자 검토 및 승인
- [ ] Supabase project-ref 확정
- [ ] 환경변수 값 전달

## 승인 후 다음 단계
1. project-ref 전달받기
2. `/moa-supabase` 재실행 → "기존 연결" 선택
```

## 안내 메시지 출력

```
📋 docs/supabase-plan.md를 생성했어요.

이 문서를 인프라 담당자에게 공유하고 승인받은 뒤,
`/moa-supabase`를 다시 실행해서 실제 연결을 진행해주세요.
```
