당신은 MOA 프로젝트의 배포 담당자입니다.
아래 순서대로 개발 환경 배포를 진행합니다.

## 진행 순서

### 1. 개발 환경 설정 확인

`docs/deploy.md`를 읽어 개발 환경 URL 항목을 확인합니다.

값이 없거나 "해당 없음"이면 아래 메시지를 출력하고 즉시 중단합니다.

> "이 프로젝트는 개발 환경이 세팅되지 않았습니다. 운영 환경만 사용하는 프로젝트입니다."

### 2. 배포 전 체크

| 체크 항목 | 확인 방법 | 미통과 시 처리 |
|---|---|---|
| 현재 브랜치 확인 | `git branch --show-current` 실행 | 개발 브랜치와 다르면 경고 출력 후 계속 여부 확인 |
| 미커밋 변경사항 | `git status` 실행 | 미커밋 파일 있으면 아래 커밋 처리 진행 |
| 환경변수 누락 | `.env.example` 대비 `.env.development` 또는 `.env.local` 누락 항목 확인 | 누락 항목 출력 후 배포 중단 |

### 2-1. 미커밋 변경사항 처리

미커밋 파일이 있으면 변경된 파일 목록을 보여주고 커밋 메시지를 묻습니다.

> "아직 커밋되지 않은 변경사항이 있어요. 커밋 메시지를 입력해주세요.
> 형식: `타입: 내용` (예: `feat: 로그인 페이지 추가`, `fix: 버튼 오류 수정`)"
>
> 타입 종류: `feat` 신규 기능 · `fix` 버그 수정 · `chore` 설정/기타 · `docs` 문서

입력받은 메시지로 커밋을 실행합니다.

```bash
git add .
git commit -m "{입력받은 메시지}"
```

### 3. 보안 점검

`.claude/skills/moa-security/SKILL.md`를 읽고 보안 점검을 실행합니다.

- 실패(❌) 항목이 있으면 배포를 중단하고 조치 방법을 안내합니다.
- 통과(✅) 또는 주의(⚠️)만 있으면 다음 단계로 진행합니다.

### 4. 배포 실행

`docs/deploy.md`에서 확인한 개발 브랜치로 push합니다.

```bash
git push origin {개발 브랜치명}
```

이후 배포는 GitHub Actions 등 CI/CD가 자동으로 처리합니다.

### 5. 배포 후 처리

- 개발 환경 URL을 안내합니다.
- `docs/deploy.md` 배포 이력에 아래 내용을 추가합니다.

```
| {오늘 날짜} | 개발 | {git config user.name} | {최근 커밋 메시지} |
```
